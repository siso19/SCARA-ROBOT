# MVC Architecture
MVC Architecture is the core design pattern of the SCARA robot server, managing system complexity by separating data, the user interface, and the coordination logic into independent layers. When the user performs an action through the interface, the Controller receives it and updates the Model's data, and the Model then propagates the change through signals so that the relevant Views reflect the new state. This unidirectional data flow ensures that the protocol state is always managed in a single, consistent location, making the code easier to maintain and extend to accommodate new experimental workflows. The Model manages only the protocol data without any knowledge of the UI or the hardware, while the View is responsible solely for visually rendering that data and forwarding user input to the Controller. The Controller sits between the two layers, acting as a coordinator that translates user actions into Model operations and reflects the updated Model state back to the View.MVC Architecture consists of Controllers, Models, and Views.
